‚ùì FAQ

Q: Can I use MySQL instead of MongoDB?
A: Technically yes, but requires:
  • Complete backend rewrite
  • Database migration
  • Schema redesign
  • 2-3 weeks of work
  • Not recommended - use MongoDB Atlas FREE instead
Q: Can I run Node.js on shared hosting?
A: No. Hostinger shared hosting doesn't support Node.js. You need:
  • VPS (paid)
  • AWS/Azure Free Tier (FREE)
  • Other cloud platforms
Q: Is hybrid deployment complicated?
A: Not really! It's just:
  • Frontend on one server (shared hosting)
  • Backend on another server (cloud)
  • Database in the cloud (MongoDB Atlas)
  • All connected via URLs/APIs
Q: What about performance?
A: Hybrid deployment performs well:
  • Frontend loads fast (shared hosting)
  • Backend on cloud (good performance)
  • Database in cloud (MongoDB Atlas)
  • Slight latency between frontend/backend (negligible)

üìã Quick Installation Steps:

  1. Connect to VPS: 
    ssh root@your-vps-ip
  2. Update system: 
    sudo apt update && sudo apt upgrade -y
  3. Install MongoDB: 
    # Import MongoDB GPG key
wget -qO - https://www.mongodb.org/static/pgp/server-7.0.asc | sudo apt-key add -

# Add MongoDB repository
echo "deb [ arch=amd64,arm64 ] https://repo.mongodb.org/apt/ubuntu jammy/mongodb-org/7.0 multiverse" | sudo tee /etc/apt/sources.list.d/mongodb-org-7.0.list

# Install MongoDB
sudo apt update
sudo apt-get install -y mongodb-org
  4. Start MongoDB: 
    sudo systemctl start mongod
sudo systemctl enable mongod
  5. Create database and user: 
    mongosh
use newsaddaindia
db.createUser({
  user: "newsadda_user",
  pwd: "your-password",
  roles: [{ role: "readWrite", db: "newsaddaindia" }]
})
  6. Enable authentication: 
    sudo nano /etc/mongod.conf
# Add: security: authorization: enabled
sudo systemctl restart mongod
  7. Update backend .env: 
    MONGODB_URI=mongodb://newsadda_user:password@localhost:27017/newsaddaindia?authSource=newsaddaindia

Storage: MongoDB data ~100MB-1GB, Logs ~100-500MB, Backups ~500MB-2GB

üîÑ GitHub CI/CD Auto-Deployment

‚úÖ Yes! You Can Automate Deployments

Set up GitHub Actions to automatically deploy to Hostinger whenever you push code to GitHub!

üöÄ Automatic Deployment Options

Option 1: Deploy to VPS (SSH)

GitHub Push ‚Üí GitHub Actions ‚Üí SSH to VPS ‚Üí Pull Code ‚Üí Restart Services

How it works:

  1. Push code to GitHub
  2. GitHub Actions triggers automatically
  3. SSH into your VPS
  4. Pull latest code
  5. Install dependencies
  6. Restart PM2 services

Option 2: Deploy to Shared Hosting (FTP/SFTP)

GitHub Push ‚Üí GitHub Actions ‚Üí Build Frontend ‚Üí FTP Upload ‚Üí Shared Hosting

How it works:

  1. Push code to GitHub
  2. GitHub Actions builds Angular app
  3. Uploads via FTP/SFTP to shared hosting
  4. Files automatically updated

üìã Quick Setup Steps:

  1. Set up GitHub Secrets:
    • Go to Repository ‚Üí Settings ‚Üí Secrets ‚Üí Actions
    • Add: HOSTINGER_VPS_HOST, HOSTINGER_VPS_USER, HOSTINGER_VPS_SSH_KEY
    • Or for FTP: HOSTINGER_FTP_HOST, HOSTINGER_FTP_USER, HOSTINGER_FTP_PASSWORD
  2. Create workflow file:
    • Create .github/workflows/deploy.yml
    • Add deployment script (see guide)
  3. Push to GitHub:
    • Every push to main branch triggers deployment
    • View deployment status in GitHub Actions tab

üí° Example Workflow (VPS):

name: Deploy to Hostinger VPS

on:
  push:
    branches: [main]

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      
      - name: Deploy Backend
        uses: appleboy/ssh-action@v0.1.7
        with:
          host: ${{ secrets.HOSTINGER_VPS_HOST }}
          username: ${{ secrets.HOSTINGER_VPS_USER }}
          key: ${{ secrets.HOSTINGER_VPS_SSH_KEY }}
          script: |
            cd ~/news-adda-backend
            git pull origin main
            npm install --production
            pm2 restart news-adda-backend

‚úÖ Benefits:

  • ‚úÖ No manual FTP/SSH uploads needed
  • ‚úÖ Automatic deployment on every push
  • ‚úÖ Deployment history and logs in GitHub
  • ‚úÖ Easy rollback capability
  • ‚úÖ Free for public repositories
  • ‚úÖ Can deploy frontend, backend, or both
